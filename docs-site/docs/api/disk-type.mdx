# DiskType

**DiskType** — справочник классов дискового хранилища: он отвечает на вопрос «какие типы
дисков доступны и в каких зонах». При создании диска вы указываете `typeId` — идентификатор
типа из этого справочника; тип задаёт характеристики хранилища (например, HDD против SSD) и
перечень зон, где он предлагается.

`DiskType` — **read-only** для пользователя: типы задаёт администратор платформы, а tenant
только читает список, чтобы выбрать подходящий `typeId` для [Disk](/api/disk). В отличие от
остальных ресурсов, id типа — человекочитаемый **литерал** (`network-ssd`), а не сгенерированный
префиксный id.

:::info Read-only на публичной поверхности
На публичном API доступны только `Get` и `List`. Создание, изменение и удаление типов —
admin-функция на cluster-internal listener (`:9091`), недоступная на external endpoint. Это
защищает каталог от изменения tenant-клиентами. Подробнее — [Авторизация](/architecture/authz).
:::

## Get

<ApiOperation method="GET" endpoint="/compute/v1/diskTypes/{diskTypeId}">

Возвращает тип диска по идентификатору-литералу.

#### Пример запроса

<CodeBlock language="bash">
  {dedent`
    curl http://localhost:18080/compute/v1/diskTypes/network-ssd \\
      -H 'Authorization: Bearer <JWT>'
  `}
</CodeBlock>

#### Пример ответа

<CodeBlock language="json">
  {dedent`
    {
      "id": "network-ssd",
      "description": "Сетевой SSD",
      "zoneIds": ["region-1-a", "region-1-b"]
    }
  `}
</CodeBlock>

<Codes codes={['invalidArgument', 'notFound', 'permissionDenied', 'internal']} />

</ApiOperation>

## Администрирование каталога

Управление типами (создание / изменение / удаление) выполняется через `InternalDiskTypeService`
на cluster-internal listener (`:9091`) — эти RPC **не публикуются** на external endpoint. Их
REST-проекция доступна только на internal mux `api-gateway` (`POST` / `PATCH` / `DELETE`
`/compute/v1/diskTypes`). Администратор задаёт `id` типа явно (это PK, immutable). Полномочия —
admin-tier; подробнее — [Авторизация](/architecture/authz).

## Сценарии использования

- **Выбор класса хранилища.** Перед `Disk.Create` прочитайте `List` и подберите `typeId` под
  требования (пропускная способность, стоимость, репликация).
- **Доступность по зонам.** `zoneIds` подсказывает, в каких зонах тип предлагается — согласуйте
  `Disk.zoneId` с этим списком.

## Подводные камни

:::caution Тип — не CRUD для tenant
Публичный API `DiskType` — только чтение. Попытка создать/изменить тип через публичную
поверхность невозможна (метода нет); управление каталогом — admin-функция на internal-порту.
:::
